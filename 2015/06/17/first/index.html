Mac+GitHub Pages+Hexo 搭建个人博客

平时经常看到一些大牛的博客都是自己搭建的，所以心血来潮自己也想整一个，了解了很多网上搭建个人博客的技术比如jekyll、Octopress等，最后选择了Hexo，网上说这个比较简单，貌似最近也比较流行。网上已经有很多文章讲解怎么使用Hexo来搭建博客了，我主要是想记录下一个小白在搭建的过程中遇到的问题。

环境准备

1、需要安装git
2、node.js
3、安装Hexo

安装git

问谷歌和度娘去吧

安装node.js

我是直接在node.js官网直接下了一个pkg包直接安装的，在Mac上面按照Hexo官网推荐的方法安装是会遇到问题的，在实际的过程中我也遇到了同样的问题：

For Mac users
You may encounter some problems when compiling. Please install Xcode from App Store first. After Xcode is installed, open Xcode and go to Preferences -> Download -> Command Line Tools -> Install to install command line tools.

根据提示我们需要先装xcode（如果这个没有那没得玩了），然后按照上面的方法我们去安装Command Line Tools工具，我的xcode是6.2的发现在Preferences下面的Download没有找到Command Line Tools这一项，网上搜才发现xcode5版本以上安装方法有变，需要我们自己通过命令 xcode-select —install安装，我们也可以去苹果开发者网上下载相应的安装包。

安装好Command Line Tools之后我们继续安装node.js,成功后我们开始安装hexo：

npm install -g hexo

通过使用上面的命令安装的时候我有遇到了另外一个错误：

Error: EACCES, mkdir ‘/usr/local/lib/node_modules/hexo’

这个问题貌似是权限不够，所以我们需要在命令前面加sudo获得管理员权限；

sudo npm install -g hexo

输入这个命令之后我发现我又遇到一个问题就是在终端的界面上面一直是转菊花的状态，从上午等到下午依然是这样，后面在一个群里问别人，别人说是因为我被墙了，无语。。。（发现奇奇怪怪的问题都我遇到了） 后面查了下我也不清楚是被墙了还是怎么，找到一个方法：

npm config set user 0
npm config set unsafe-perm true

这样设置之后继续安装hexo 终于成功了
详细的node.js和hexo安装过程

mardown写文章并同步到git

在安装成功之后我们接下来就是写文章，因为hexo的功能就是能将markdown文件最终生成html静态文件，这里推荐一个markdown编辑器。

配置_config.yml文件
里面的元素节点代表什么意义我们可以参考上一篇博文，这里我主要讲下我遇到的问题。我通过命令生成了静态文件后准备把网站部署到git上面去，我们需要配置我们自己的git地址到_config.yml的Deploy节点下面去：

deploy:
type: git
repo: git@github.com:OldGhost366/OldGhost366.github.io.git

我在这里遇到的问题是我在type和repo的冒号后面没有加空格，然后我不管我怎么输入hexo deploy 都不能把我的网站同步到git上面去，也不会报错，搞了一下午还是没成功，然后在一个群里：287306637 把我的配置文件的源码给他们看，群里的哥们开始也没有发现问题，后面终于有一个哥们提示我说没有加空格。我倒。。。 就这样把我折腾了好久，所以大家切记在_config.yml里面的节点后面的冒号一定要加上一个空格，如果忘记的话你会发现有很多奇葩的问题出现， 切记！！！！
